# java基础-注解机制

# 注解基础

# 注解定义

从jdk1.5开始，java引入了注解，提供了一种为代码添加元数据的方法。可以对 包、类、接口、字段、方法参数、局部变量等进行注解

元数据解释(百度百科open in new window):元数据（Metadata），又称中介数据、中继数据，为描述数据的数据（data about data），主要是描述数据属性（property）的信息，用来支持如指示存储位置、历史数据、资源查找、文件记录等功能。

# 内置注解

  • @Override ：表示当前的方法是重写父类的方法。
  • @Deprecated：标记表示不推荐使用（过时）的代码。
  • @SuppressWarnings：抑制编译器警告。

代码理解：

class ATest {

    public int test(){
        return 1;
    }
}

class BTest extends ATest{

    /**
     * 重写父类的test方法
     * @return
     */
    @Override
    public int test() {
        return 2;
    }

    /**
     * 过时的方法-不推荐使用
     */
    @Deprecated
    public void oldTest(){

    }

    /**
     * 抑制编译器告警
     * @return
     */
    @SuppressWarnings("rawtypes")
    public List get(){
        List<String> list = new ArrayList<>();
        return list;
    }
}

进一步查看内置注解类

# 内置注解-@Override
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.SOURCE)
public @interface Override {
}

@Target(ElementType.METHOD) 表示该注解可以修饰的是方法，@Retention(RetentionPolicy.SOURCE) 表示该注解只在编译时有效，编译后的 .class 文件中是不存在的。这个注解主要用于重写父类方法时标注，编译器会对此做检查，如果发现父类中不存在这个方法，或者和父类方法中的名称不同，则报错。

# 内置注解-@Deprecated
@Documented
@Retention(RetentionPolicy.RUNTIME)
@Target(value={CONSTRUCTOR, FIELD, LOCAL_VARIABLE, METHOD, PACKAGE, PARAMETER, TYPE})
public @interface Deprecated {
}

@Target(value={CONSTRUCTOR, FIELD, LOCAL_VARIABLE, METHOD, PACKAGE, PARAMETER, TYPE}) 表示该注解可以修饰 构造函数、字段、枚举的常量、局部变量、方法、包、参数、接口、类、枚举、注解，@Retention(RetentionPolicy.RUNTIME) 表示该注解在编译时和编译后(运行时) 都会保留。这个注解主要是标明已过时，提醒用户不要再使用。

# 内置注解-@SuppressWarnings
@Target({TYPE, FIELD, METHOD, PARAMETER, CONSTRUCTOR, LOCAL_VARIABLE})
@Retention(RetentionPolicy.SOURCE)
public @interface SuppressWarnings {
    String[] value();
}

@Target({TYPE, FIELD, METHOD, PARAMETER, CONSTRUCTOR, LOCAL_VARIABLE}) 表示该注解可以修饰 构造函数、字段、枚举的常量、局部变量、方法、参数、接口、类、枚举、注解，@Retention(RetentionPolicy.SOURCE) 表示该注解只在编译时有效，编译后的 .class 文件中是不存在的。这个注解的作用是告诉编译器忽略指定的告警信息，可以取的值是：

参数作用
all抑制所有警告
boxing抑制装箱、拆箱操作时候的警告
cast抑制映射相关的警告
dep-ann抑制启用注释的警告
deprecation抑制过期方法警告
fallthrough抑制确在switch中缺失breaks的警告
finally抑制finally模块没有返回的警告
hiding抑制与隐藏变数的区域变数相关的警告
incomplete-switch忽略没有完整的switch语句
nls忽略非nls格式的字符
null忽略对null的操作
rawtype使用generics时忽略没有指定相应的类型
restriction抑制与使用不建议或禁止参照相关的警告
serial忽略在serializable类中没有声明serialVersionUID变量
static-access抑制不正确的静态访问方式警告
synthetic-access抑制子类没有按最优方法访问内部类的警告
unchecked抑制没有进行类型检查操作的警告
unqualified-field-access抑制没有权限访问的域的警告
unused抑制没被使用过的代码的警告

# 元注解

元注解是用于定义注解的注解，包括@Target、@Retention、@Inherited、@Documented、@Repeatable。

元注解也是Java自带的标准注解，只不过用于修饰注解，比较特殊。

# 元注解-@Target

@Target用来表示注解作用范围，超过这个作用范围，编译的时候就会报错。

public enum ElementType {
    // 类、接口、枚举类
    TYPE, 
    // 成员变量（包括：枚举常量）
    FIELD, 
    // 成员方法
    METHOD, 
    // 方法参数
    PARAMETER, 
    // 构造方法
    CONSTRUCTOR, 
    // 局部变量
    LOCAL_VARIABLE, 
    // 注解类
    ANNOTATION_TYPE, 
    // 可用于修饰：包
    PACKAGE, 
    // 类型参数，JDK 1.8 新增
    TYPE_PARAMETER, 
    // 使用类型的任何地方，JDK 1.8 新增
    TYPE_USE 
}
# 元注解-@Retention

来表示注解的生命周期，生命周期的长短取决于@Retention的属性RetentionPolicy指定的值

public enum RetentionPolicy {
    // 源文件保留
    SOURCE,    
    // 编译期保留，运行时删除，默认值
    CLASS,    
    // 运行期保留，可通过反射去获取注解信息  
    RUNTIME   
}

具体可以看下面的列表

取值描述作用范围使用场景
RetentionPolicy.SOURCE表示注解只保留在源文件，当java文件编译成class文件，就会消失源文件只是做一些检查性的操作，，比如 @Override 和 @SuppressWarnings
RetentionPolicy.CLASS注解被保留到class文件，但jvm加载class文件时候被遗弃，这是默认的生命周期class文件（默认）要在编译时进行一些预处理操作，比如生成一些辅助代码（如 ButterKnife）
RetentionPolicy.RUNTIME注解不仅被保存到class文件中，jvm加载class文件之后，仍然存在运行时也存在需要在运行时去动态获取注解信息
# 元注解-@Documented

Documented注解的作用是：描述在使用 javadoc 工具为类生成帮助文档时是否要保留其注解信息。

如下代码在使用javadoc工具生成文档时会显示 @DocumentedMy 注解信息

@Documented
public @interface DocumentedMy {
    String value() default "测试文档";
}

@DocumentedMy
class MyTest{
    public String getStr(){
        return "123";
    }
}
# 元注解-@Inherited

@Inherited注解：被该注解修饰的注解具有继承性。一个类使用了被 @Inherited 修饰的注解，那么其子类将自动具有该注解

使用测试代码来验证一下：

  • 定义一个注解用@Inherited修饰
@Inherited
@Target({ElementType.TYPE})
@Retention(RetentionPolicy.RUNTIME)
public @interface TestInherited {
    String value();
}
  • 定义父类和子类(FatherInherited和ChildInherited)，打印子类拥有的注解以及名称
@TestInherited(value = "测试继承性")
public class FatherInherited {

}
class ChildInherited extends FatherInherited{

    public static void main(String[] args) {
        Class clazz = ChildInherited.class;
        Annotation[] annotations = clazz.getAnnotations();
        System.out.println("子类的注解个数："+annotations.length);
        for (Annotation annotation : annotations) {
            System.out.println(annotation.annotationType().getSimpleName());
        }
    }
}

结果：

子类的注解个数：1
TestInherited

通过上面的代码测试，可以看到 子类并没有显示的使用@TestInherited注解,但是子类继承了父类，而父类是有该注解的，所以子类也有会拥有该注解。

# 元注解-@Repeatable(java8)

主要解决在在类，属性或者方法中多次使用同一个注解；@Repeatable 会让可读性更强。

# JDK8之前

java8之前，重复性注解的解决方式：

public @interface Role {
    String role();
}

@interface Roles{
    Role[] value();
}

class TestRoleOld{
    @Roles({@Role(role = "Admin"),@Role(role = "Manager")})
    public void doing(){
    }
}
# JDK8之后

在Role上使用@Repeatable 指向 Roles 就可以在方法上使用如下方式来重复使用注解：

@Repeatable(Roles.class)
public @interface Role {
    String role();
}

@interface Roles{
    Role[] value();
}
class TestRoleNew{
    @Role(role = "Admin")
    @Role(role = "Manager")
    public void doing(){
    }
}
# 元注解-@Native (Java8)

使用 @Native 注解修饰成员变量，则表示这个变量可以被本地代码引用，常常被代码生成工具使用。对于 @Native 注解不常使用，了解即可

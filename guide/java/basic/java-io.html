# 文件概述

什么是文件：

文件，我们并不陌生,文件是保存数据的地方，比如大家经常使用的 word文档，txt文档，excel文档 等等。它既可以保存一张图片，也可以保存视频，声音。。。

文件流

文件在程序中是以流的形式来操作的

流：数据在数据源(文件) 和程序(内存) 之间经历的路径
输入流:数据从数据源(文件)到程序(内存)的路径
输出流:数据从程序(内存)到数据源(文件)的路径

# 常用的文件操作

创建文件的三种方式

// 方式1  new File(文件路径)

获取文件的相关信息

# IO流

  • I/O 是Input/Output 的缩写， I/O 技术是非常实用的技术，用户处理数据传输。如：读写文件，网络通信等。
  • java程序中，对于数据的输入/输出操作以“ 流(stream)” 的方式进行。
  • java.io包下提供了各种“流”类和接口，用以获取不同种类的数据，并通过方法输入或输出数据

流的分类

  • 按操作数据单位不同分为：字节流(8bit),字符流(按字符)
  • 按数据流的流向不同分为:输入流，输出流
  • 按流的角色的不同分为：节点流，处理流/包装流
(抽象基类)字节流字符流
输入流InputStreamReader
输出流OutputStreamWriter

InputStream，OutputStream，Reader，Writer 都是抽象类，要想使用只能创建其子类对象。
java的IO流共设计40多个类，实际上非常规则，都是从如上4个抽象基类派生的。
由这4个类派生出来的自雷名称都是以其父类名作为子类名后缀。

# IO流体系-常用的类

# InputStram 字节输入流

  1. InputStream抽象类是所有类字节输入流的超类
  2. InputStream 常用的子类
    • FileInputStream:文件输入流
    • BufferedInputStream：缓冲字节输入流
    • ObjectInputStream：对象字节输入流

FileInputStream 案例

// 读取文件
@Test
public void readFile01(){
    String filePath = "D:\\hello.txt";
    int readLine = 0;
    byte[] bytes = new byte[1024];
    // 放在 try() 里面的流，会自动关闭，不需要finally
    try (FileInputStream fileInputStream = new FileInputStream(filePath))
    {
        while ((readLine = fileInputStream.read(bytes)) != -1){
            System.out.println(new String(bytes,0,readLine));
        }
    } catch (Exception e) {
        e.printStackTrace();
    }
}

# OutputStream 字节输出流

FileOutputStream 案例

@Test
public void writeFile(){
    String filePath = "D:\\helloWrite.txt";
    // 放在 try() 里面的流，会自动关闭，不需要finally
    try (
            // new FileOutputStream(filePath) 创建方式，会覆盖原有的内容
            // new FileOutputStream(filePath,true);这种创建方式，是在原有的取追加
            FileOutputStream fileOutputStream = new FileOutputStream(filePath)
    )
    {
        String str = "hello,world";
        fileOutputStream.write(str.getBytes());
    } catch (Exception e) {
        e.printStackTrace();
    }
}

案例2-复制图片

@Test
public void copyPng(){
    String intPng = "D:\\test\\xiaoxin.jpeg";
    String outPng = "D:\\test\\xiaoxin1.jpeg";
    try(
            FileInputStream fileInputStream = new FileInputStream(intPng);
            FileOutputStream fileOutputStream = new FileOutputStream(outPng)

        byte[] buf = new byte[1024];

        }

    }catch (Exception e){
        e.printStackTrace();
    }
}

# FileReader 字符输入流

FileReader相关方法

  1. new FileReader(File/String)
  2. read：每次读取单个字符，返回该字符，到文件末尾返回1
  3. read(char[]):批量读取多个字符到数组，返回读取到的字符数，到文件末尾返回-1

FileReader案例

// 读取文件
@Test
public void ReaderFile1(){
    String filePath = "D:\\hello.txt";
    int readLen = 0;
    char[] cbuf = new char[1024];
    try (FileReader fileReader = new FileReader(filePath))
    {
        while ((readLen = fileReader.read(cbuf)) != -1){
            System.out.println(new String(cbuf,0,readLen));
        }
    } catch (Exception e) {
        e.printStackTrace();
    }
}

# FileWriter 字符输出流

FileWriter相关方法

  1. new FileWriter(File/String); 覆盖模式，相当于流的指针在首端
  2. new FileWriter(File/String,true);追加模式，相当于流的指针在尾端
  3. write(int):写入单个字符
  4. write(char[]):写入指定数组
  5. write(char[],off,len):写入指定数组的指定部分
  6. write(string):写入整个字符串
  7. write(string,off,len):写入字符串的指定部分
  • String类：toCharArray-将String转换成char[]

FileWriter使用后，必须关闭(close)或刷新(flush),否则写入不到指定的文件！

FileWriter案例

@Test
public void WriterFile(){
    String filePath = "D:\\hello.txt";
    // 追加模式，不加true是覆盖模式
    try (FileWriter fileWriter = new FileWriter(filePath,true))
    {
        // 写入单个字符
        fileWriter.write("H");
        // 写入数组
        fileWriter.write("测试".toCharArray());
        // 写入字符串
        fileWriter.write("你好");
        // 写入字符串的部分--只会写进去浙江
        fileWriter.write("浙江杭州",0,2);
        fileWriter.close();
    } catch (Exception e) {
        e.printStackTrace();
    }
}

# 节点流和处理流

  1. 节点流可以从一个特定的数据源读写数据

  2. 处理流(包装流) 是连接在已存在的流之上，为程序提供更为强大的读写功能

  3. 示意图

  4. 两者的区别

    • 节点流是底层流/低级流，直接跟数据源相接
    • 处理流包装节点流，既可以消除不同节点流的实现差异，也提供了更加方便的方法来完成输出输出
    • 处理流对节点流进行包装，使用了修饰器设计模式，不会直接与数据源进行连接

  5. 处理流的优点

    • 性能的提高:主要以增加缓冲的方式来提高输入输出的效率
    • 操作的便捷：处理流提供了一系列便捷的方法来一次输入输出大批量的数据，使用更加灵活方便

# 缓冲流

BufferedReader

字符输入处理流，按照字符来读取数据，关闭时只需要关闭外层流即可。

案例

public static void main(String[] args) {
    String str  = "D:\\hello.txt";
    try (BufferedReader bufferedReader = new BufferedReader(new FileReader(str))){
        String readLine;
        // 按行读取
        while ((readLine = bufferedReader.readLine()) != null){
            System.out.println(readLine);
        }
    } catch (Exception e) {
        e.printStackTrace();
    }
}

BufferedWriter

字符输出处理流，按照字符来读取数据，关闭时只需要关闭外层流即可。

案例

public static void main(String[] args) {
    String str  = "D:\\hello.txt";
    // 没有关闭流，是因为使用了 try-with-resources
    try(
        // 追加形式
        BufferedWriter bufferedWriter = new BufferedWriter(new FileWriter(str,true))
    ){
        bufferedWriter.write("hello，测试");
        bufferedWriter.newLine();
        bufferedWriter.write("hello1，测试");
        bufferedWriter.newLine();
        bufferedWriter.write("hello2，测试");
    }catch (Exception e){
        e.printStackTrace();
    }
}

拷贝文件案例

public static void main(String[] args) {
    String strRead  = "D:\\hello.txt";
    String strWriter  = "D:\\hello2.txt";
    try(
        BufferedReader bufferedReader  = new BufferedReader(new FileReader(strRead));
        BufferedWriter bufferedWriter = new BufferedWriter(new FileWriter(strWriter))
    ){
        String readLine;
        // readLine 读取一行，但是不会带上换行
        while ((readLine = bufferedReader.readLine()) != null){
            bufferedWriter.write(readLine);
            // 换行
            bufferedWriter.newLine();
        }
    }catch (Exception e) {
        e.printStackTrace();
    }
}

注意

BufferedReader和BufferedWriter 是按照字符操作
不要去操作二进制文件(音频，视频),会造成文件损坏

BufferedInputStream

字节输入处理流，按照字节来读取数据，关闭时只需要关闭外层流即可。

BufferedOutputStream

字节输出处理流，按照字节来读取数据，关闭时只需要关闭外层流即可。

BufferedInputStream和BufferedOutputStream 拷贝文件案例

public static void main(String[] args) {
    String strIn = "D:\\test\\xiaoxin.jpeg";
    String strOut = "D:\\test\\xiaoxin2.jpeg";
    try (
        BufferedInputStream bufferedInputStream = new BufferedInputStream(new FileInputStream(strIn));
        BufferedOutputStream bufferedOutputStream = new BufferedOutputStream(new FileOutputStream(strOut))
    ){
        byte[] buf = new byte[1024];
        int readerLine = 0;
        while ((readerLine = bufferedInputStream.read(buf)) != -1){
            bufferedOutputStream.write(buf,0,readerLine);
        }
    } catch (Exception e) {
        e.printStackTrace();
    }
}

注意

BufferedInputStream和BufferedOutputStream 既可以处理二进制文件，同时也可以处理字符信息文件。

ObjectOutputStream、ObjectInputStream

序列化和反序列化

  1. 序列化就是在保存数据时，保存数据的值和数据的类型
  2. 反序列化就是在恢复数据时，恢复数据的值和数据的类型
  3. 需要让某个对象支持序列化机制，则必须让其类是可序列化的，为了让某个类是可序列化的，必须实现如下两个接口之一：
    • Serializable （标记接口）
    • Externalizable

ObjectOutputStream 序列化案例

public static void main(String[] args) {
    String strOut = "D:\\test\\people.txt";
    try (ObjectOutputStream oos = new ObjectOutputStream(new FileOutputStream(strOut))){
        // int -> Integer (实现了 Serializable)
        oos.writeInt(100);
        //boolean -> Boolean (实现了 Serializable)
        oos.writeBoolean(true);
        //char -> Character (实现了 Serializable)
        oos.writeChar('a');
        //double -> Double (实现了 Serializable)
        oos.writeDouble(9.5);
        //String (实现了 Serializable)
        oos.writeUTF("测试测试");
        People people = new People("测试", 22, "男");
        oos.writeObject(people);
    } catch (IOException e) {
        e.printStackTrace();
    }
}

ObjectInputStream 反序列化案例

public static void main(String[] args) {
        String strOut = "D:\\test\\people.txt";

        try (ObjectInputStream ois = new ObjectInputStream(new FileInputStream(strOut))
        ){
            // 读取数据，读取数据需要和保存时的顺序一致
            System.out.println(ois.readInt());
            System.out.println(ois.readBoolean());
            System.out.println(ois.readChar());
            System.out.println(ois.readDouble());
            System.out.println(ois.readUTF());
            Object o = ois.readObject();
            System.out.println(o.getClass());
        } catch (Exception e) {
            e.printStackTrace();
        }
    }

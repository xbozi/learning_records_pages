# JAVA基础-泛型

# 什么是泛型？

Java泛型是Java编程语言中的一个特性，它允许在编码时使用类型参数，这样可以让代码更加通用和可复用。泛型编程意味着编写的代码可以被不同类型的对象使用，同时还能在编译时提供类型安全检查。

# 为什么使用泛型？

泛型的本质是为了参数化类型（在不创建新的类型的情况下，通过泛型指定的不同类型来控制形参具体限制的类型）。也就是说在泛型使用过程中，操作的数据类型被指定为一个参数，这种参数类型可以用在类、接口和方法中，分别被称为泛型类、泛型接口、泛型方法。

  • 举一个例子：
List arrayList = new ArrayList();
arrayList.add("aaaa");
arrayList.add(100);

for(int i = 0; i< arrayList.size();i++){
    String item = (String)arrayList.get(i);
    Log.d("泛型测试","item = " + item);
}

上述代码运行，就会抛出类型转换异常

java.lang.ClassCastException: java.lang.Integer cannot be cast to java.lang.String

因为ArrayList可以存放任意类型，先存入了String类型的，接着存入了 Integer 再输出的时候要强转String，所以运行异常了。
我们期望在编译阶段就被告知错误，而不是再运行才知道； 泛型应运而生

我们将上述代码的第一行改为如下，编译器就会在编译阶段告知我们错误了

//List 中只能放String
List<String> arrayList = new ArrayList<String>();
...
//arrayList.add(100); 在编译阶段，编译器就会报错

# 泛型的使用

泛型有三种使用方式，分别为：泛型类、泛型接口、泛型方法

# 泛型类

  • 简单的泛型类
//此处T可以随便写为任意标识，常见的如T、E、K、V等形式的参数常用于表示泛型
public class People<T> {
    private T value;
    public T getValue() {
        return value;
    }
    public void setValue(T value) {
        this.value = value;
    }
    public People(T value) {
        this.value = value;
    }
    public People() {
    }

}
class PeopleDemo{
    public static void main(String[] args) {
        //在实例化泛型类时，指定T的具体类型,这里指定的是 String类型
        People<String> people= new People<>();
        people.setValue("泛型测试"); //设置字符串
        System.out.println(people.getValue()); //输出字符串 泛型测试
    }
}

上述定义的泛型类在实例化时必须要传入泛型类型吗？ 并不是这样的，在实例化时传入泛型类型，则根据传入的类型做相应的限制，这时泛型才会起限制作用。如果不传入泛型类型，那么 T 就可以为任意值

People people1= new People(1);
People people2= new People(1.1);
People people3= new People(false);
System.out.println(people1.getValue());//输出1
System.out.println(people2.getValue());//输出1.1
System.out.println(people3.getValue());//输出false
  • 多元泛型
//指定了两个泛型类型
public class MultiplePeople<K,V>{
    private K key;
    private V value;
    public K getKey() {
        return key;
    }
    public void setKey(K key) {
        this.key = key;
    }
    public V getValue() {
        return value;
    }
    public void setValue(V value) {
        this.value = value;
    }
    public MultiplePeople(K key, V value) {
        this.key = key;
        this.value = value;
    }
    public MultiplePeople() {
    }
}

class MultiplePeopleDemo{
    public static void main(String[] args) {
        //实例化时 传入两个类型 String和Integer
        MultiplePeople<String, Integer> demo = new MultiplePeople<>();
        demo.setKey("小博"); //第一个类型只能是String
        demo.setValue(20); //第二个类型只能是Integer
        System.out.println(demo.getKey());
        System.out.println(demo.getValue());
    }

# 泛型接口

//定义泛型接口
public interface Info<T> {
    public T getValue();
}
/**
 * 未传入泛型实参时，与泛型类的定义相同，在声明类的时候，需将泛型的声明也一起加到类中
 * @param <T>
 */
class InfoImpl<T> implements Info<T>{
    private T value;
    @Override
    public T getValue() {
        return this.value;
    }
    public void setValue(T value) {
        this.value = value;
    }
    public InfoImpl(T value) {
        this.value = value;
    }
}
/**
 * 在实现类实现泛型接口时，如已将泛型类型传入实参类型，则所有使用泛型的地方都要替换成传入的实参类型
 */
class InfoImplValue implements Info<String>{
    @Override
    public String getValue() {
        return null;
    }
}
class InfoImplDemo{
    public static void main(String[] args) {
        // 通过子类实例化对象
        Info<String> info = new InfoImpl<>("内容"); 
        System.out.println(info.getValue()); //输出内容
    }
}

# 泛型方法

泛型方法是比较复杂的

/**
 * 泛型方法的基本介绍
 * @param tClass 传入的泛型实参
 * @return T 返回值为T类型
 * 说明：
 *     1）public 与 返回值中间<T>非常重要，可以理解为声明此方法为泛型方法。
 *     2）只有声明了<T>的方法才是泛型方法，泛型类中的使用了泛型的成员方法并不是泛型方法。
 *     3）<T>表明该方法将使用泛型类型T，此时才可以在方法中使用泛型类型T。
 *     4）与泛型类的定义一样，此处T可以随便写为任意标识，常见的如T、E、K、V等形式的参数常用于表示泛型。
 */
public <T> T genericMethod(Class<T> tClass)throws InstantiationException ,
  IllegalAccessException{
        T instance = tClass.newInstance();
        return instance;
}

调用泛型方法

//此时 obj 的类型为Test
Object obj = genericMethod(Class.forName("com.test.Test"));

说明：

  1. 定义泛型方法时必须在返回值前加一个 <T> 用来声明这是一个泛型方法；持有一个泛型T，才能返回T作为方法的返回值
  2. Class<T> 的作用就是指明泛型的具体类型，
  3. 泛型方法要求的参数是 Class<T> 类型的，Class.forName() 返回值就是 Class<T>，因此可以作为入参；在上面的例子中 forName() 传入的是test的完整路径，所以 Class<T> 为 Class<Test> ;泛型方法返回的也是 Class<Test>,其中T 被指明为 Test。

# 泛型的上下界

  • 一个例子 先看下面的代码：
class A {
}
class B extends A {
}
class demo{
    // 如下两个方法不会报错
    public static void funA(A a) {
    }
    public static void funB(B b) {
        funA(b);
    }
    
    public static void funC(List<A> a) {
    }
    // D 方法会报错
    public static void funD(List<B> b) {
        funC(b);
    }
}

上面的代码 funD方法会报错，因为 B 是 A 的子类，但是List<B> 不是 List<A> 的子类，在类型转换时会编译器会报错。为了解决这一问题；泛型中引入了上下界，上界使用 <? extends A>表示该参数类型可以是A(上边界)或者A的子类类型；编译时会擦除到类型A。

public static void funC(List<? extends A> a) {
}
//这样就不会报错了
public static void funD(List<B> b) {
    funC(b);
}
  • 泛型上下界引入 上界(extends)：
public class GenericUp<T extends Number> {
    private T var;
    public T getVar() {
        return var;
    }
    public void setVar(T var) {
        this.var = var;
    }
    public GenericUp() {
    }
}
class GenericUpDemo{
    public static void main(String[] args) {
        GenericUp<Integer> genericUp = new GenericUp<Integer>();
        genericUp.setVar(111);
        System.out.println(genericUp.getVar());//111
    }
}

上述代码中 Number 就是上界，传入的值只能是 Number或者Number的子类，Integer是Number的子类，所以运行正常，输出 111

下界(super):

public class GenericDown <T>{
    private T var;
    public T getVar() {
        return var;
    }
    public void setVar(T var) {
        this.var = var;
    }
    public GenericDown() {
    }
}
class GenericDownDemo{
    public static void main(String[] args) {
        //声明String的泛型对象
        GenericDown<String> stringGenericDown = new GenericDown<>();
        //声明Object的泛型对象
        GenericDown<Object> objectGenericDown = new GenericDown<>();
        stringGenericDown.setVar("111");
        objectGenericDown.setVar(234);
        run(stringGenericDown);
        run(objectGenericDown);
    }
    /**
     * 只能接收 String，或着String的父类；Sting的父类只有Object
     * @param genericDown
     */
    public static void run(GenericDown<? super String> genericDown){
        System.out.println(genericDown.getVar());
    }
}

上述代码中<? super String> 代表只能接收String或者String的父类;

总结(拷贝的@pdai大佬的,原文链接：https://pdai.tech/md/java/basic/java-basic-x-generic.html)

<?> 无限制通配符
<? extends E> extends 关键字声明了类型的上界，表示参数化的类型可能是所指定的类型，或者是此类型的子类
<? super E> super 关键字声明了类型的下界，表示参数化的类型可能是指定的类型，或者是此类型的父类

// 使用原则《Effictive Java》
// 为了获得最大限度的灵活性，要在表示 生产者或者消费者 的输入参数上使用通配符，使用的规则就是：生产者有上限、消费者有下限
1. 如果参数化类型表示一个 T 的生产者，使用 < ? extends T>;
2. 如果它表示一个 T 的消费者，就使用 < ? super T>；
3. 如果既是生产又是消费，那使用通配符就没什么意义了，因为你需要的是精确的参数类型。

# java为什么被称为伪泛型？

Java泛型这个特性是从JDK 1.5才开始加入的，因此为了兼容之前的版本，Java泛型的实现采取了"伪泛型"的策略，即Java在语法上支持泛型，但是在编译阶段会进行所谓的"类型擦除"（Type Erasure），将所有的泛型表示（尖括号中的内容）都替换为具体的类型（其对应的原生态类型），就像完全没有泛型一样。
